KEY FINDINGS (5)
================================================================================

1. MANUAL PHASE 4 IS BIGGEST BOTTLENECK
   - Current: Create folder, move files, update MEMORY.md, commit → 15 min/analysis
   - At scale: 50 analyses/year × 15 min = 750 min (12.5 hours) wasted on setup
   - Fix: `/analyze` skill automates folder creation, metadata, git commit

2. NO DEPENDENCY GRAPH = SILENT BREAKAGE RISK
   - If Golf_Analytics columns change, no way to auto-detect which scripts break
   - Current: 168 scripts, no way to know which use which columns
   - Fix: Dependency parser + `/dependency-check` skill flags impacted scripts

3. METADATA SPARSE = RESULTS UNTRUSTWORTHY
   - "How many combos tested?" (in CSV but not metadata)
   - "Was this tested on 2025-2026 data?" (unclear from filename)
   - "Is this analysis still valid?" (no re-run date recorded)
   - Fix: METADATA.yaml template + auto-filled by analysis process

4. SIGNAL DEPLOYMENT UNTRACKED = LIVE BET RISK
   - 3 approved signals in production but no deployment ledger
   - No "when deployed?", "current performance?", "re-validate date?" tracking
   - Risk: Signal drifts for weeks, no one notices
   - Fix: SIGNAL_DEPLOYMENT.md ledger + `/signal-deploy` skill + weekly review

5. SCRIPTS LACK VERSIONING = UNCLEAR WHICH TO USE
   - 168 scripts in SCRIPTS/, mixed naming (analyze_*, AUDIT_*, system*_*)
   - No status indicator (experimental? deprecated? production-stable?)
   - User can't tell which script is "the one"
   - Fix: Script metadata header (status, version, last_tested)
