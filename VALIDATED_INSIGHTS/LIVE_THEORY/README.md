# LIVE THEORY — Untested Intuitions

**Status:** 🔵 Intuited or channeled; awaiting test design
**Purpose:** Hold emerging theories before they're formally tested
**Owner:** User (provides theories) + Claude (structures them)

---

## What Goes Here

Theories that are:
- **Intuited** (you've thought deeply about a pattern)
- **Channeled** (you've received insight through meditation/channeling)
- **Pattern-observed** (you noticed something interesting in data)
- **Theory-informed** (grounded in color rhythm / numerology / divination knowledge)

But NOT YET tested formally.

---

## How to Add a Theory

1. **Tell Claude** the theory in conversation
2. **Claude creates** file: `[theory_name]_[YYYYMMDD].md`
3. **Claude fills** template with your input
4. **You review** and approve

Example conversation:
> "I've intuited that Purple players tend to close stronger. It aligns with Purple's grounding energy. Want to test it?"

Claude creates: `purple_players_close_stronger_20260404.md`

---

## Template (What Each File Contains)

```markdown
---
title: [Theory Name]
status: LIVE_THEORY
confidence: 1
date_created: YYYY-MM-DD
last_updated: YYYY-MM-DD
---

## Theory Statement
One clear sentence describing the claim.

## Origin
Where this came from (intuition, channeling, observation, theory).

## Reasoning
Why you believe it (color traits, divination logic, observed patterns).

## Rough Prediction
What would support or contradict this?

## Notes
Anything else relevant.

## Next Steps
- [ ] Design test plan
- [ ] Run analysis
- [ ] Review results
```

## What Happens Next

Once a theory is here, you decide: **test now or explore more?**

- **Test now:** Claude designs test → moves to NEEDS_TESTING
- **Explore more:** Keep in LIVE_THEORY, refine theory, test later
- **Abandon:** Move to REJECTED with note "User retraction"
